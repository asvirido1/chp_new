# Testing Smoke

Короткий ручной runbook для текущего dev/staging состояния `ЧПОК`.

## Перед стартом

Нужно, чтобы были подняты:

- backend на `http://127.0.0.1:8080`
- admin на `http://localhost:5173`
- mobile/Expo приложение, если тест идёт через пользовательский UI

Для admin:

- открой `http://localhost:5173`
- введи staging/dev секрет
- перейди в раздел `Reports`

Если в ходе сценария создаётся новый `ЧПОК`, потом его удобно искать в админке по:

- времени создания
- описанию
- провайдеру

## 5-Minute Checklist

Если нужен самый быстрый продуктовый прогон после небольших изменений, достаточно пройти это:

1. Открыть `admin`, ввести staging/dev секрет, убедиться, что админка открывается.
2. Создать один `ЧПОК` без медиа и убедиться, что он появился в `Reports`.
3. Создать один `ЧПОК` с фото и убедиться, что в detail page есть image preview.
4. Загрузить одно видео в существующий `ЧПОК` и убедиться, что в detail page есть video player.
5. В detail page изменить статус и проверить новую запись в `Истории статусов`.
6. Добавить заметку и проверить её появление в `Заметках модератора`.
7. Для фото и видео нажать `Открыть оригинал` и убедиться, что оба URL открываются.

Если все 7 пунктов проходят, короткий smoke для текущего состояния `ЧПОК` считается зелёным.

## Сценарий 1. Пользователь создаёт чпок без медиа

Шаги:

1. В mobile открыть экран создания жалобы.
2. На шаге фото выбрать `Пропустить`.
3. Выбрать категорию.
4. Выбрать сервис.
5. Ввести описание нарушения.
6. Отправить жалобу.

Успех:

- пользователь видит экран `Жалоба отправлена!`
- в admin появляется новый `ЧПОК`
- у жалобы `mediaCount = 0`

Где смотреть результат:

- mobile success screen
- admin `Reports`
- admin detail page конкретного обращения

## Сценарий 2. Пользователь создаёт чпок с фото

Шаги:

1. В mobile открыть экран создания жалобы.
2. Сделать фото или выбрать фото из галереи.
3. Пройти остальные шаги создания жалобы.
4. Отправить жалобу.
5. Открыть созданное обращение в admin detail page.

Успех:

- жалоба создаётся
- фото реально загружается в Storage
- в admin detail page видно image preview
- ссылка `Открыть оригинал` открывает изображение

Где смотреть результат:

- mobile success screen
- admin detail page, блок `Медиа`
- при необходимости: Supabase Storage bucket `report-media`

## Сценарий 3. Пользователь создаёт чпок с видео

Шаги:

1. Подготовить короткое тестовое видео.
2. Загрузить видео через текущий media flow.
   Для ручного smoke в текущем окружении можно использовать upload route:

```sh
curl -i -X POST \
  -F "mediaType=video" \
  -F "file=@/path/to/test-video.mp4;type=video/mp4" \
  "http://127.0.0.1:8080/api/reports/<REPORT_ID>/media/upload"
```

3. Открыть тот же `REPORT_ID` в admin detail page.

Успех:

- upload route отвечает `201 Created`
- в `report_media` появляется запись с `mediaType = video`
- в admin detail page виден video player
- ссылка `Открыть оригинал` открывает `.mp4`

Где смотреть результат:

- HTTP response upload route
- admin detail page, блок `Медиа`
- при необходимости: Supabase Storage bucket `report-media`

## Сценарий 4. Модератор открывает чпок в админке

Шаги:

1. Открыть `http://localhost:5173`
2. Пройти admin guard.
3. Перейти в `Reports`.
4. Найти нужный `ЧПОК`.
5. Открыть detail page.

Успех:

- без секрета админка не открывается
- после ввода секрета открывается список обращений
- detail page загружается без ошибок

Где смотреть результат:

- admin login screen
- admin list page
- admin detail page

## Сценарий 5. Модератор меняет статус

Шаги:

1. Открыть detail page обращения.
2. В блоке `Изменить статус` выбрать новый статус.
3. При необходимости добавить комментарий.
4. Нажать `Сохранить статус`.

Успех:

- UI показывает успешное обновление
- текущий статус меняется
- в блоке `История статусов` появляется новая запись

Где смотреть результат:

- badge текущего статуса в detail page
- блок `История статусов`

## Сценарий 6. Модератор добавляет заметку

Шаги:

1. Открыть detail page обращения.
2. В блоке `Добавить заметку` ввести текст.
3. Нажать `Добавить заметку`.

Успех:

- заметка сохраняется без ошибки
- в блоке `Заметки модератора` появляется новая запись

Где смотреть результат:

- блок `Заметки модератора` в admin detail page

## Сценарий 7. Модератор проверяет media preview

Шаги:

1. Открыть detail page обращения, где уже есть media.
2. Проверить все карточки в блоке `Медиа`.
3. Для фото проверить preview.
4. Для видео проверить, что виден player с controls.
5. Для одной-двух карточек нажать `Открыть оригинал`.

Успех:

- фото рендерится как изображение
- видео рендерится как video player
- `Открыть оригинал` открывает рабочий URL
- нет broken legacy preview от старых `example.com` записей

Где смотреть результат:

- admin detail page, блок `Медиа`
- отдельная вкладка браузера с оригинальным файлом

## Быстрый regression checklist

После изменений, которые затрагивают `reports`, `media`, `admin` или `storage`, достаточно быстро пройти:

1. Создать `ЧПОК` без медиа.
2. Создать `ЧПОК` с фото.
3. Загрузить видео в существующий `ЧПОК`.
4. Открыть оба обращения в admin.
5. Проверить image preview и video preview.
6. Изменить статус одного обращения.
7. Добавить заметку.

Если все 7 пунктов проходят, текущий product smoke для `ЧПОК` считается зелёным.
